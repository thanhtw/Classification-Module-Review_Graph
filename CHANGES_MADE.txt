================================================================================
DATA STORAGE STRUCTURE
================================================================================

After training, predictions and labels are stored as:

results/modular_multimodel/model_artifacts/
├── bert/
│   ├── fold_1/
│   │   ├── predictions.npy (240, 3)     ← NEW: Per-fold predictions
│   │   ├── labels.npy (240, 3)          ← NEW: Per-fold labels
│   │   ├── pytorch_model.bin            (existing)
│   │   ├── config.json                  (existing)
│   │   └── ...
│   ├── fold_2/ through fold_10/
│   ├── all_folds_predictions.npy (2398, 3)  ← NEW: Aggregated (run aggregate script)
│   └── all_folds_labels.npy (2398, 3)       ← NEW: Aggregated (run aggregate script)
├── lstm/, bilstm/, roberta/, ...
├── linear_svm/, logistic_regression/, naive_bayes/
├── cnn_attention/
└── llm_few_shot/, llm_zero_shot/

Similar structure for all 10 models.

================================================================================
DATA FORMAT SPECIFICATION
================================================================================

Shape:
  - predictions.npy: (n_samples, 3) - binary multilabel predictions
  - labels.npy: (n_samples, 3) - true binary multilabel targets

Data type:
  - dtype: np.int64 or int (values are 0 or 1)

Columns (order):
  - Column 0: relevance (0 or 1)
  - Column 1: concreteness (0 or 1)
  - Column 2: constructive (0 or 1)

Per-fold:
  - For 10-fold CV: ~240 test samples per fold
  - Shape per fold: (240, 3) approximately

Aggregated:
  - All folds combined: 2398 test samples (full dataset)
  - Shape aggregated: (2398, 3)

Example prediction array (first 5 rows):
  [[0, 1, 0],  # Sample 1: not relevant, concrete, not constructive
   [1, 1, 1],  # Sample 2: relevant, concrete, constructive
   [1, 0, 1],  # Sample 3: relevant, not concrete, constructive
   [0, 0, 0],  # Sample 4: none of the labels apply
   [1, 1, 0]]  # Sample 5: relevant, concrete, not constructive

================================================================================
WORKFLOW
================================================================================

Step 1: Train all models
   Command: python scripts/train.py --n_folds 10
   Output: predictions.npy and labels.npy saved in each fold directory
   
   Automatic:
   - Each model saves predictions and labels after evaluation
   - No additional configuration needed
   - Happens for every fold

Step 2: Aggregate predictions (optional but recommended)
   Command: python scripts/aggregate_predictions.py
   Output: all_folds_predictions.npy and all_folds_labels.npy
   
   Function:
   - Loads all fold-specific predictions
   - Combines into single array across all folds
   - Saves aggregated files for easier analysis

Step 3: Generate confusion matrices
   Command: python scripts/generate_confusion_matrices.py
   Input: Uses saved predictions and labels (no model loading needed)
   Output: Confusion matrix visualizations
   
   Ready:
   - Can use generated predictions directly
   - No need to reload models
   - Fast generation without model inference

Step 4: Verify workflow (optional)
   Command: python verify_predictions_workflow.py
   Output: Verification report
   
   Checks:
   - Predictions saved correctly
   - Data format correct
   - Aggregated files present
   - All steps completed successfully

================================================================================
USAGE EXAMPLES
================================================================================

Load predictions for one model:
  import numpy as np
  preds = np.load('results/modular_multimodel/model_artifacts/bert/fold_1/predictions.npy')
  labels = np.load('results/modular_multimodel/model_artifacts/bert/fold_1/labels.npy')
  print(f"Shape: {preds.shape}")  # (240, 3) approximately

Load aggregated predictions:
  import numpy as np
  preds = np.load('results/modular_multimodel/model_artifacts/bert/all_folds_predictions.npy')
  labels = np.load('results/modular_multimodel/model_artifacts/bert/all_folds_labels.npy')
  print(f"Shape: {preds.shape}")  # (2398, 3)

Generate confusion matrix:
  from sklearn.metrics import confusion_matrix
  cm = confusion_matrix(labels[:, 0], preds[:, 0])  # For 'relevance' label
  print(cm)

Compute metrics from saved predictions:
  from sklearn.metrics import accuracy_score, precision_score, recall_score, f1_score
  acc = accuracy_score(labels, preds)
  prec = precision_score(labels, preds, average='micro')
  recall = recall_score(labels, preds, average='micro')
  f1 = f1_score(labels, preds, average='micro')

================================================================================
FILE SIZES
================================================================================

Per-fold files (for each fold):
  - predictions.npy: ~8 KB (240 samples × 3 labels × 8 bytes per int64)
  - labels.npy: ~8 KB

Aggregated files:
  - all_folds_predictions.npy: ~77 KB (2398 samples × 3 labels × 8 bytes)
  - all_folds_labels.npy: ~77 KB

Total per model:
  - 10 folds × 16 KB = 160 KB
  - + aggregated files = 314 KB per model
  - × 10 models = ~3.14 MB for all predictions

Storage impact: Minimal (~3 MB for complete 10-model system)
